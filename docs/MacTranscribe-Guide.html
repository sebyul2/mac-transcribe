1. 설치와 첫 실행

  1. MacTranscribe.dmg를 열고 Mac Transcribe.app을 /Applications로 드래그합니다.
  2. 첫 실행 시 권한 안내 창이 뜹니다. 세 가지 권한이 필요합니다:
    • 마이크 — 받아쓰기·녹음
    • 입력 모니터링 — 트리거 키(⌃Fn 등) 감지
    • 화면 기록 — 시스템 오디오(통화·영상 소리) 캡처 시에만
  3. 권한을 켜면 앱 재시작 없이 즉시 동작합니다. 메뉴바에 🎤 아이콘이 나타납니다.

2. 기본 사용법 — 트리거 키와 메뉴

메뉴바 아이콘 클릭 시 — 자주 쓰는 항목만 남긴 단축 메뉴

  • Start / Stop Recording — 잠금 녹음 토글. 녹음 중엔 메뉴바 아이콘이 ⏺로 바뀝니다.
  • Live Translation — 체크되어 있으면 잠금 녹음이 통역 세션으로 시작됩니다.
  • Transcript Window — 실시간 전사/번역 로그 창.
  • Open Saved Folder — 저장 폴더(~/Documents/MacTranscribe)를 Finder로 엽니다.

3. 실시간 동시통역

통역은 잠금 녹음 + Live Translation 체크 조합으로 동작합니다:

  1. 메뉴에서 Live Translation을 체크
  2. Settings ▸ Translation에서 출발어/도착어와 오디오 소스 선택
  3. Settings ▸ Engine ▸ Translation에서 번역 엔진 선택 (아래 4절)
  4. ⌃⇧Fn으로 녹음 시작 → 화면 하단에 번역 자막이 흐릅니다

자막 읽는 법

다른 결정이 아니었을까라고 말했습니다.
그리고 6일, 발로군 선수는 벨기에와의 경기에

화면 하단 자막 — 흰색은 확정, 회색은 아직 바뀔 수 있는 부분

  • 흰색 = 확정된 번역. 다시 바뀌지 않습니다.
  • 회색 = 말이 진행 중이라 아직 다듬어지는 중인 부분.
  • 번역이 아직 도착하지 않았거나 실패한 줄은 원문이 회색으로 표시됩니다 — 번역 엔진이 죽어도 자막이 비지 않습니다.
  • 최대 3줄이 흐르고, 완성된 줄은 위로 밀려 올라갈 뿐 읽는 중에 사라지지 않습니다.
전사 언어 = 발화 언어. 일본어 회의를 들으려면 인식 언어도 日本語여야 합니다. Translation 탭에서 출발어를 고르면 인식 언어가 자동으로 따라갑니다.

Apple Translation (온디바이스)

맥에 내장된 번역 엔진입니다. 설정이 전혀 필요 없고 인터넷 없이 동작하며, LLM 계정이 없어도 통역이 됩니다. 처음 쓰는 언어쌍은 시스템이 모델 다운로드 승인을 요청합니다. 세션 시작 직후 1~3초의 준비 시간 동안 들어온 문장은 준비가 끝나는 순간 몰아서 번역됩니다.

LLM

Engine ▸ Meeting에 설정된 계정(ChatGPT 구독 OAuth 또는 OpenAI API 키)을 사용하되, 번역용 모델은 따로 고를 수 있습니다. 로컬 전사 결과를 문장 단위로 재번역하며, 이전 번역을 재사용해 자막 흔들림을 줄입니다.

DeepL Voice (스트리밍) — 최고 품질

오디오를 DeepL 서버로 직접 스트리밍합니다. 전사·문장 분절·번역을 전부 DeepL이 수행하므로 로컬 전사의 한계(어미 잘림, 문장 파편화)가 원천적으로 없습니다.

  1. Engine ▸ Translation ▸ Provider를 DeepL Voice (streaming)로 선택
  2. 같은 행에 나타나는 DeepL API Key 입력 (⌘V 붙여넣기 가능)
  3. Test 클릭 — 키·유료플랜·Voice 권한을 한 번에 검증하고, 성공하면 확인 음성이 재생됩니다. 검증된 키는 ●●●●로 가려집니다 (클릭하면 다시 편집 가능)
자동 복구. 네트워크가 끊기거나 세션이 만료되면 자동 재연결됩니다. 밀린 오디오는 최근 3초만 유지하고 버립니다 — 통역은 항상 "현재"를 따라갑니다. 무음 구간에도 세션이 유지되어(keepalive) 소리가 돌아오면 즉시 재개됩니다.

5. 번역 음성 읽기(TTS)와 볼륨 덕킹

Settings ▸ Translation의 두 체크박스:

  • Read translations aloud (TTS) — 확정된 번역을 시스템 음성으로 읽어줍니다. 짧은 조각은 잠시 모았다가 한 호흡으로 이어 읽어 동시통역처럼 자연스럽고, 문장 중간에 절대 끊지 않습니다. 대화가 음성보다 빨라지면 오래된 내용은 건너뛰고 따라잡습니다.
  • Duck other audio while speaking — 통역 음성이 나오는 동안 원본 소리(시스템 볼륨)를 절반으로 낮추고, 통역 음성은 거의 원래 크기로 얹습니다. 실제 통역 부스 피드처럼 원음 위에 통역이 들립니다. 음성이 멈추면 1초쯤 뒤 볼륨이 복원됩니다.

6. 회의 모드와 자동 회의록

Live Translation이 꺼진 상태의 잠금 녹음은 회의 모드입니다: 실시간 자막 + 전사 파일 저장.

  • Settings ▸ Meeting ▸ Audio source — 마이크 또는 시스템 오디오(화상회의 소리). 통역용 소스와 별개입니다.
  • Generate meeting notes — 녹음 종료 후 회의록(참석자·논의·결정·액션아이템)을 Markdown으로 자동 생성합니다.

Engine 탭 — Meeting과 Translation이 서로 다른 엔진을 쓸 수 있습니다

  • Meeting Provider가 LLM이면 Model·계정 행이, Claude Code (CLI)면 CLI 설치 상태가 표시됩니다.
  • Translation Provider가 LLM이면 번역 전용 Model 선택이, DeepL Voice면 API Key + Test가 같은 자리에 나타납니다. Apple이면 아무 설정도 필요 없습니다.

8. 저장 폴더 구조

폴더내용

9. 문제 해결 — 상태 메시지 사전

메시지의미와 대처
Downloading … model…해당 언어의 인식/번역 모델을 처음 내려받는 중. 최초 1회만 걸립니다(수 초~수십 초).
Recognition stalled — restarting…음성은 들리는데 인식 결과가 멎어 인식기를 자동 재구축하는 중. 몇 초 내 재개됩니다. 캡처·백업은 계속 돕니다.
Audio capture stalled — restarting…시스템 오디오 캡처(SCStream)가 조용히 죽어 자동 재시작하는 중. macOS의 알려진 현상으로, 앱이 스스로 복구합니다.
Voice reconnecting (n)…DeepL Voice 세션이 끊겨 재연결 중. 밀린 과거는 버리고 현재부터 재개합니다.
On-device translation unavailable for this pairApple 번역이 해당 언어쌍을 지원하지 않거나 모델 다운로드가 거절됨. 자막은 원문으로 계속 표시됩니다.
Generating meeting notes…회의록 작성 중(수 분). 앱을 종료하지 마세요 — 이 동안은 덮개를 닫아도 잠들지 않습니다.
붙여넣기(⌘V)가 안 됨구버전 이슈 — 최신 버전으로 업데이트하세요. 모든 설정 필드에서 ⌘C/⌘V/⌘X/⌘A가 동작합니다.
통역 품질이 이상할 때 1순위 점검: 인식(Recognition) 언어가 실제 발화 언어와 같은지 확인하세요. 한국어 인식기로 일본어를 들으면 깨진 전사가 번역기로 넘어갑니다. Translation 탭에서 출발어를 다시 고르면 자동으로 맞춰집니다. (DeepL Voice는 전사를 서버가 하므로 이 문제가 없습니다.)
